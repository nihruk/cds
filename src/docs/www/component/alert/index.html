Alerts are status messages shown to the user, embedded in a page, and sometimes but not always about that page.

Intended behaviour

Common colours for these messages are green for success, yellow/orange/amber for warning, red for error, and blue for info. These colours are merely a convention, and we must choose our own colours and shades that communicate intent and are accessible to all users.

Additionally, to facilitate colourblind users, alerts must have a second visual identifier that communicates the messageâ€™s intent with more than mere colour (shape or text).

When to use this component

Common statuses include success to inform about a user action that completed successfully, warning to tell users that they should be aware of something, error for user or server actions that went wrong and could not be completed, or info for messages that are informational but not otherwise important.
